# RESHAPE.md — wpr-permit-tracker (Instance Zero)

Handover spec for Claude Code. Everything in here was validated against real data
(May 2026 report, 838/838 records parsed, zero field anomalies) in a recon session
on 2026-07-02. Do not re-derive what is already measured. Do not add fallbacks.

## What this is

A monthly building-permit tracker for Wausau Pilot & Review. Pipeline:

```
City of Wausau monthly PDF  →  adapter  →  ledger (idempotent merge)  →  policy filter
→  geocode  →  public JSON  →  React/Vite map widget  →  WordPress iframe
```

Instance Zero serves Wausau + Schofield + Rib Mountain (one source PDF covers all
three — Wausau Inspections contracts to neighboring municipalities). The repo is
built config-driven from day one (Gavel lessons): all jurisdiction, branding, and
sponsor knowledge lives in `instance.json`. Nothing in `web/` or `pipeline/` may
name a city.

Product cadence is **monthly**, matching the source. There is no real-time mode.
Editorial frame: "Monthly Development Report presented by [sponsor]."

## Source facts (measured, do not rediscover)

- Reports page: `https://www.wausauwi.gov/your-government/inspections/permit-information/building-permit-activity`
  → yearly folders → monthly PDF links via `showpublisheddocument/{id}/{ticks}`.
  Archives back to 2015. ~1 month publication lag.
- **Akamai fronts wausauwi.gov.** Plain requests get 403. `curl_cffi` with
  `impersonate="chrome"` returned 200 from a datacenter IP in recon. `fetch.py`
  uses curl_cffi only. If Actions IPs get blocked later, telemetry will show it;
  the fix at that point is Webshare residential proxies (known pattern from
  wpr-obituaries) — do NOT pre-build proxy support.
- PDF: landscape 792×612, generated by the "Evolve" permit system, template
  name "Monthly - Permits Issued". May 2026 = 79 pages, 838 records.

### Font protocol (section structure)

| Element              | Font                | Size |
|----------------------|---------------------|------|
| Report title         | Arial Bold          | 14.3 |
| **Jurisdiction header** | **Arial Bold**   | **10.5** |
| Column headers       | Arial Bold          | 8.8  |
| Field labels (Sq Ft…)| Arial Bold          | 7.8  |
| Data                 | Arial regular       | 8.1  |
| Summary lines        | Arial regular       | 8.9  |

A line whose words are ALL Bold with 10 < size < 11 is a jurisdiction header.
Records appearing before the first such header have `jurisdiction: "unassigned"`
(Evolve's null-jurisdiction group sorts first; May had 2 address-less paving
records there).

### Column boundaries (measured x0 ranges, points)

```
permit_id    25–100      owner       400–520
template    100–190      contractor  520–635
address     190–313      issue_date  635–710
parcel_id   313–400      last_insp   710–792
```

Parcel tokens start at exactly x0=314 (97/97 sampled). The 313 boundary is
correct; 316 was tried and wrong. Do not "round to header positions" — data
columns do not align with header x0s.

### Record grammar

```
<header line>        permit_id template address parcel owner contractor dates
<wrap lines 0..n>    continuation text, assigned to columns by x0
<sqft line>          "Finished Sq Ft [n] Unfinished Sq Ft [n] Units [n] Description [text]"
<wrap lines 0..n>    description continuation — EXCEPT see contractor-wrap defect
```

- New record trigger: first word matches `^20\d{7}$` AND sits in the permit_id
  column. Permit IDs are YYYYMM + sequence → natural `source_permit_id`.
- Line grouping MUST be tolerance-based clustering (3pt on `top`), never
  `round(top)` — baselines differ by fractions of a point and rounding splits
  lines, which was the root cause of every field-bleed defect in recon.
- The sqft line is parsed positionally (see adapter `_parse_sqft_line`);
  missing labels or unclassifiable words raise. This is the layout-drift
  tripwire.

### RESOLVED DEFECTS (fixed and covered by tests/acceptance.py)

1. Contractor/owner wraps below the sqft line: fixed positionally. Description
   data always starts at x0=459 (measured); post-sqft lines starting 400-455
   split word-by-word between owner (<520) and contractor (>=520); lines
   starting >=459 and <520 are description; >=520 are contractor. Verified on
   the Chamber of Commerce record (owner "WI PUBLIC SERVICE CORP ATTN: TAX
   DEPARTMENT", contractor "GREATER WAUSAU CHAMBER OF COMMERCE").
2. April 2026 revealed wraps can share the SAME visual line as the sqft
   labels. The sqft line is now parsed positionally (bold words = labels,
   numeric words in label-bounded windows = values, other words routed by
   column). The regex approach is gone; do not reintroduce it.
3. The bare "Valuation" label (no $) that opens each section summary block
   flushes the current record.

Known accepted ambiguity: a contractor wrap sharing the sqft line exactly is
indistinguishable from description flow. Not observed in Apr/May 2026.

### Summaries

- `"{Jurisdiction} Permits Issued N"` per section; the FINAL "Wausau Permits
  Issued 838" line is the GRAND total (2+5+11+820=838). Parse accordingly.
- Valuation exists ONLY as per-jurisdiction totals
  (`"{Jurisdiction} Valuation $N"`). May 2026: Wausau $6,166,797, Schofield
  $969,425, Rib Mountain $350,901. There is NO per-permit valuation — the
  contract has no valuation field. Do not add one.

## PermitRecord contract (contract.py)

```python
@dataclass(frozen=True)
class PermitRecord:
    source_permit_id: str      # "202604958"
    jurisdiction: str          # "wausau" | "schofield" | "rib_mountain" | "unassigned"
    template: str              # Evolve template, verbatim: "Com Building", "Excavation", …
    address: str | None
    parcel_id: str | None      # 12–15 digits or None
    owner: str | None
    contractor: str | None
    issue_date: str            # ISO 8601 date
    last_insp: str | None      # ISO 8601 date
    finished_sqft: int | None
    unfinished_sqft: int | None
    units: int | None
    description: str
    source: str                # "wausau_monthly_pdf"
    source_document: str       # PDF url or filename
    as_of: str                 # report month, "2026-05"
    verified: bool             # True only if run count == report grand total
```

Validation throws on: malformed permit id, parcel not matching `^\d{12,15}$`,
unparseable dates. `verified` is set for the whole batch only when parsed record
count equals the report's grand total; a mismatched batch fails the run.

## Ledger (ledger.py)

Same discipline as wpr-obituaries: merge on `source_permit_id`, append-only.
Re-running a month is a no-op. A changed record for an existing id raises
(the monthly report is a snapshot; silent mutation means something is wrong).
Geocode results (`lat`, `lon`, `geocode_status`) are stored ON the ledger record
so geocoding is paid once per permit, ever.

## Geocoding (geocode.py)

Census Bureau geocoder (free, no key):
`https://geocoding.geo.census.gov/geocoder/locations/onelineaddress` with
`benchmark=Public_AR_Current`, address = `"{address}, WI"`. One correct path —
no Nominatim fallback, no Google. Unresolved → `lat: null`, `geocode_status:
"unmatched"`; record renders in list view, not on map. Rate: batch politely,
~1 req/sec is fine at 838/month.

## policy.py (editorial rules — Shereen signs off on thresholds)

Rules are data in `instance.json` (`policy` block); policy.py enforces them:

1. **Drop** records with no address (unmappable, includes the "unassigned" group).
2. **Aggregate, never pin:** templates in `aggregate_templates`
    (default `["Excavation", "Plumbing Lateral"]` — May had 363 + 190 of these,
    overwhelmingly the city lead-service-line program) collapse into a
    street-level program layer: counts per street, NO owner names, no
    individual pins. This is both the noise filter and the privacy rule.
3. **Feature layer:** templates in `feature_templates` (default
    `["Com Building", "Residential Building"]`) are the newsworthy pins —
    39 records in May.
4. **Residential privacy:** for any template NOT in feature/aggregate sets,
    suppress `owner` when it looks like a person rather than an org (heuristic:
    contains no token from `ORG_TOKENS = {LLC, INC, CORP, CO, TRUST, CHURCH,
    CITY, VILLAGE, COUNTY, SCHOOL, …}`). Suppression means the field is absent
    from the PUBLIC JSON — the ledger keeps everything.
5. Everything else publishes with standard fields.

## Web widget (web/)

React 18 + Vite, Leaflet + OpenStreetMap tiles (no key, no alternates).
Reads `permits.json`, `summary.json`, `instance.json`. Knows zero jurisdictions.

- Map: pins colored by template class (feature = accent teal, other = ink);
  aggregate layer rendered as street-level count markers, toggleable.
- Detail card: template, address, contractor, issue date, sqft/units,
  description, "View property history" link when `parcel_id` joins to
  wpr-property-transactions (URL pattern in instance.json; omit link otherwise).
- List view under the map: filter by jurisdiction, template class, month.
- Summary strip: month's valuation totals per jurisdiction + permit count.
- Sponsor slot renders only when `sponsor.zone_id` is set.
- WordPress embed via iframe, standard WPR pattern.

## telemetry.py

One JSON line per run: `{ts, report_month, fetch_status, http_status, bytes,
records_parsed, grand_total, verified, new_records, geocode_calls,
geocode_unmatched, duration_s}`. Ten lines of code. This is the COGS ledger for
Gavel-style pricing later — it ships in v1.

## monthly.yml (Actions)

Cron monthly (run twice mid-month, e.g. the 12th and 19th, since publication lag
varies) →

- `fetch.py`: scrape the current-year folder page for the newest PDF link; if
  no NEW report month is found, exit 0 with telemetry line `fetch_status:
  "no_new_report"` (that is the one legitimate soft exit)
- parse → validate (count must equal grand total or FAIL the job)
- merge → geocode new records → policy → build → commit data + deploy Pages

## Acceptance (definition of done)

1. May 2026 PDF parses 838/838, `verified: true`, zero parcel/date anomalies,
   contractor-wrap defect fixed (spot-check the Chamber of Commerce record).
2. Re-running May is a ledger no-op.
3. April 2026 PDF (one folder up) parses with count == its grand total —
   proves the format holds across months. If it doesn't, the run fails loudly
   and the failure message names the first non-conforming line.
4. Widget renders 39 feature pins, aggregate layer, summary strip, and detail
   cards from real data with zero hardcoded jurisdiction strings in web/ or
   pipeline/ (grep check: "wausau" appears only in the adapter filename,
   adapter internals, and instance.json).
5. telemetry/runs.jsonl has a line per run.
